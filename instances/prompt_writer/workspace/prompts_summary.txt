# AI搜索系统提示词集合

## 概述

本文档包含为三层AI搜索系统设计的三个核心智能体提示词。该系统专门用于开源情报搜集和分析，采用分层协作的方式完成复杂的搜索和分析任务。

## 智能体架构

### 1. 主智能体 (Master Agent)
**文件**: `master_agent_prompt.txt`
**角色**: 开源情报搜索大师与协调者

**核心职责**:
- 深入的需求分析和确认
- 制定全面的搜索策略
- 协调多个子智能体的工作
- 实时调整搜索方向
- 综合分析所有搜索结果
- 协调写作智能体生成报告

**关键特性**:
- 强大的思辨和分析能力
- 基于结果的适应性调整
- 全力以赴不轻易放弃
- 开源情报专家的思维模式

**工具限制**:
- 只能通过Task工具调用子智能体
- 不能直接使用搜索工具
- 主要读取和分析，不直接修改内容

### 2. 子智能体 (Sub-agent)
**文件**: `sub_agent_prompt.txt`
**角色**: 搜索执行者与工具专家

**核心职责**:
- 执行主智能体分配的搜索任务
- 熟练使用5个专业搜索工具
- 高效获取和整理信息
- 将结果保存为结构化文件
- 返回文件路径给主智能体

**可用工具**:
1. `searxng_search` - 多引擎并行搜索
2. `analyze_pages` - 深度内容分析
3. `download_resources` - 资源下载管理
4. `save_screenshot` - 网页截图保存
5. `run_browser_task` - 复杂交互任务

**输出要求**:
- 文件结构：原文目录 + 归纳目录 + 其他分类
- 只返回文件路径
- 注重实质信息而非形式

### 3. 写作智能体 (Writing Agent)
**文件**: `writing_agent_prompt.txt`
**角色**: 内容综合者与专业报告撰写者

**核心职责**:
- 基于搜索结果撰写专业报告
- 逐章节生成高质量内容
- 进行信息的交叉验证
- 整合所有章节形成完整报告
- 确保内容的准确性和专业性

**工作特点**:
- 严格基于事实，禁止编造
- 逐章节创作，追求高质量
- 完成后整体优化
- 只返回最终文件路径

## 系统工作流程

### 阶段1：需求分析（主智能体）
1. 接收用户查询
2. 深度提问澄清需求
3. 将确认需求保存为文件
4. 制定搜索策略

### 阶段2：搜索执行（子智能体）
1. 主智能体通过Task工具调用子智能体
2. 子智能体执行具体搜索任务
3. 使用专业工具获取信息
4. 保存结果为结构化文件
5. 返回文件路径

### 阶段3：迭代优化（主智能体）
1. 分析搜索结果
2. 识别信息空白
3. 调整搜索策略
4. 继续调用子智能体
5. 重复直到信息完整

### 阶段4：报告生成（写作智能体）
1. 主智能体协调写作智能体
2. 写作智能体分析所有搜索结果
3. 逐章节撰写报告内容
4. 整合优化形成最终报告

## 核心设计原则

### 1. 职责分离
- 主智能体：策略和协调
- 子智能体：执行和收集
- 写作智能体：整合和输出

### 2. 迭代优化
- 基于结果持续改进
- 动态调整搜索方向
- 不满足于初步结果

### 3. 质量保证
- 每个智能体都有明确的质量标准
- 多层次的质量检查机制
- 专业水准的内容输出

### 4. 文件驱动
- 所有关键信息保存为文件
- 确保可追溯性和可复现性
- 便于智能体间的信息传递

## 使用指南

### 部署步骤
1. 将三个提示词分别配置到对应的智能体
2. 确保子智能体可以访问所需的搜索工具
3. 建立智能体间的通信机制（Task工具）
4. 设置文件存储和共享机制

### 关键配置
- 子智能体需要5个搜索工具的访问权限
- 主智能体需要调用Task工具的能力
- 写作智能体需要访问搜索结果文件的权限
- 所有智能体共享文件存储空间

### 最佳实践
1. **需求确认**: 主智能体要充分澄清用户需求
2. **搜索策略**: 制定多层次、多维度的搜索计划
3. **信息验证**: 重要信息要进行交叉验证
4. **迭代改进**: 持续优化搜索策略直到满足需求
5. **质量把控**: 每个环节都要严格质量控制
