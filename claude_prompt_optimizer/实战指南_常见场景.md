## 进阶技巧

### 技巧 1: 链式提示词（Prompt Chaining）

**概念**：将复杂任务拆分成多个步骤，每步使用不同的提示词。

**示例**：内容创作流水线
```python
# 步骤 1: 生成大纲
prompt_1 = generate_prompt("Create a detailed outline for a blog post", ["TOPIC"])

# 步骤 2: 扩展每个部分
prompt_2 = generate_prompt("Write a detailed section based on an outline point",
                           ["OUTLINE_POINT", "WORD_COUNT"])

# 步骤 3: 润色和优化
prompt_3 = generate_prompt("Improve readability and flow of a text",
                           ["DRAFT_TEXT"])
```

**优势**：
- 每个步骤专注于单一任务
- 更容易调试和优化
- 可以重用步骤

---

### 技巧 2: 动态示例选择

**概念**：根据任务类型自动选择相关示例。

**实现**：
```python
def enhance_with_examples(prompt_template, task_type):
    """根据任务类型添加相关示例"""

    examples = {
        "classification": """
        <example>
        Input: "This product is amazing! Best purchase ever."
        Output: <category>Positive</category><confidence>95</confidence>
        </example>
        """,

        "summarization": """
        <example>
        Input: [300-word article]
        Output: [3-sentence summary capturing key points]
        </example>
        """,

        "extraction": """
        <example>
        Input: "John Doe, CEO of Acme Corp, announced..."
        Output: {
          "person": "John Doe",
          "title": "CEO",
          "organization": "Acme Corp"
        }
        </example>
        """
    }

    if task_type in examples:
        return prompt_template + examples[task_type]

    return prompt_template
```

### 技巧 5: 上下文压缩

**问题**：长文档会消耗大量 tokens。

**解决方案**：两阶段处理
```python
# 阶段 1: 提取相关部分（使用 Haiku，便宜）
compress_prompt = """
From the following document, extract only the parts relevant to: {$QUERY}

Document: {$LONG_DOCUMENT}

Output only the relevant excerpts.
"""

relevant_parts = call_claude(compress_prompt, model="haiku")

# 阶段 2: 详细分析（使用 Sonnet，准确）
analysis_prompt = """
Based on these relevant excerpts, {$DETAILED_TASK}

Excerpts: {$RELEVANT_PARTS}
"""

final_result = call_claude(analysis_prompt, model="sonnet")
```

**成本节省**：可减少 50-80% 的 token 使用。

## 问题排查

### 问题 1: 输出格式不一致

**症状**：
- 有时包含 XML 标签，有时不包含
- 字段顺序随机变化

**解决方案**：
```python
# ❌ 模糊的指令
"Provide your answer with reasoning"

# ✅ 明确的格式要求
"""
You MUST output in this exact format:

<reasoning>
[Your step-by-step reasoning here]
</reasoning>

<answer>
[Your final answer here]
</answer>

Do not include any text outside these tags.
"""
```

---

### 问题 2: 幻觉（编造信息）

**症状**：
- AI 提供不在源文档中的信息
- 引用不存在的内容

**解决方案**：
```python
# 强制引用源文本
prompt_template = """
Rules:
1. ONLY use information from {$DOCUMENT}
2. If the answer is not in the document, say "I cannot answer this based on the provided document"
3. For every claim, include a direct quote from the document

Example:
Claim: The company revenue was $10M
Quote: "Our annual revenue reached $10 million" (from Section 2)
"""
```

---

### 问题 3: 过于冗长或过于简短

**症状**：
- 回答要么太短缺少细节，要么太长包含无关信息

**解决方案**：
```python
# 明确长度和详细程度
prompt_template = """
Response length: {$LENGTH_REQUIREMENT}
- "brief": 2-3 sentences
- "moderate": 1-2 paragraphs (100-150 words)
- "detailed": 3-4 paragraphs (300-400 words)

Detail level: {$DETAIL_LEVEL}
- "high-level": Main points only
- "standard": Main points + supporting details
- "comprehensive": Main points + details + examples + implications
"""
```

---

### 问题 4: 忽略边界条件

**症状**：
- 空输入时崩溃
- 超长输入时输出不完整
- 特殊字符导致格式错误

**解决方案**：
```python
# 添加边界条件处理
prompt_template = """
Before processing, check:

1. If {$INPUT} is empty or only whitespace:
   Output: "No input provided. Please provide [description of expected input]."

2. If {$INPUT} exceeds 10,000 characters:
   Output: "Input too long. Please provide a shorter input (max 10,000 characters)."

3. If {$INPUT} contains invalid characters or format:
   Output: "Invalid input format. Expected format: [description]."

Only proceed with normal processing if all checks pass.
"""
```

## 性能优化

### 优化 1: 缓存常用模板

**问题**：每次都调用 Metaprompt 生成提示词浪费时间和成本。

**解决方案**：
```python
import json
from pathlib import Path

class PromptCache:
    def __init__(self, cache_dir="prompt_cache"):
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(exist_ok=True)

    def get(self, task, variables):
        """从缓存获取提示词"""
        cache_key = self._make_key(task, variables)
        cache_file = self.cache_dir / f"{cache_key}.json"

        if cache_file.exists():
            with open(cache_file) as f:
                return json.load(f)["prompt"]

        return None

    def set(self, task, variables, prompt):
        """保存提示词到缓存"""
        cache_key = self._make_key(task, variables)
        cache_file = self.cache_dir / f"{cache_key}.json"

        with open(cache_file, "w") as f:
            json.dump({
                "task": task,
                "variables": variables,
                "prompt": prompt
            }, f, indent=2)

    def _make_key(self, task, variables):
        """生成缓存键"""
        import hashlib
        content = f"{task}:{','.join(sorted(variables))}"
        return hashlib.md5(content.encode()).hexdigest()

# 使用
cache = PromptCache()

def get_or_generate_prompt(task, variables):
    """先从缓存获取，没有则生成"""
    cached = cache.get(task, variables)
    if cached:
        print("📦 Using cached prompt")
        return cached

    print("🔨 Generating new prompt")
    prompt = generate_prompt(task, variables)
    cache.set(task, variables, prompt)
    return prompt
```

### 优化 2: 批量处理

**问题**：逐个处理多个输入效率低。

**解决方案**：
```python
def batch_process(prompt_template, inputs, batch_size=5):
    """批量处理输入"""

    results = []

    for i in range(0, len(inputs), batch_size):
        batch = inputs[i:i+batch_size]

        # 构造批量提示词
        batch_prompt = f"""
        Process the following {len(batch)} items according to these instructions:

        {prompt_template}

        Items to process:
        """

        for idx, item in enumerate(batch, 1):
            batch_prompt += f"\n<item_{idx}>\n{item}\n</item_{idx}>\n"

        batch_prompt += """
        Output format:
        <results>
        <result_1>[Result for item 1]</result_1>
        <result_2>[Result for item 2]</result_2>
        ...
        </results>
        """

        # 处理批次
        response = call_claude(batch_prompt)
        results.extend(parse_batch_results(response))

    return results
```

**效果**：可提升 2-3 倍吞吐量。

### 优化 4: 模型选择策略

**决策树**：
```python
def select_model(task_complexity, required_quality, budget):
    """根据需求选择模型"""

    if task_complexity == "simple":
        return "claude-haiku-4-5"  # 快速、便宜

    elif task_complexity == "medium":
        if required_quality == "high":
            return "claude-sonnet-4-5"  # 平衡
        else:
            return "claude-haiku-4-5"  # 节省成本

    else:  # complex
        if budget == "unlimited":
            return "claude-opus-4-5"  # 最强
        else:
            return "claude-sonnet-4-5"  # 性价比高

# 使用
model = select_model(
    task_complexity="medium",
    required_quality="high",
    budget="limited"
)
```
